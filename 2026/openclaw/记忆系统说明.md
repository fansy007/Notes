# 爱因斯坦记忆系统说明

> 本系统定义爱因斯坦（QClaw 助手）的记忆存储与检索规则，适用于飞书实例。

---

## 存储结构

```
Obsidian vault: /Users/hg26502/ObsidianNotes/
~/.qclaw/workspace/
├── MEMORY.md                 # 长期记忆核心库（每次对话必读）
├── memory/
│   ├── YYYY-MM-DD.md         # 每日原始日志（当天对话结束时自动写入）
│   └── archive/
│       ├── 2026-01-index.md  # 归档索引（每月1号自动生成）
│       ├── 2026-02-index.md
│       ├── 2026-01-03.md     # 归档后的原始文件
│       └── ...
```

---

## 各层文件职责

### 1. MEMORY.md（长期记忆核心库）

- 每次对话开始时必读
- 内容：海宁的背景偏好、重要决策结论、系统规则（番茄钟制度等）、常见坑
- 容量目标：永久保持在几百行以内
- 更新时机：每次你说"晚安"或`/new`时自动更新

### 2. memory/YYYY-MM-DD.md（每日日志）

- 当天对话结束时自动写入
- 内容：今日关键讨论、决定、任务、问题
- 篇幅：几百字，只记重要的，不写流水账
- 文件命名：`YYYY-MM-DD`，中文注释写在文件内

### 3. memory/archive/（归档存储）

- 触发时机：每月1号自动归档超过90天的 daily 文件
- 归档同时生成当月 index 索引文件（YYYY-MM-index.md）
- index 内容：每条日志的日期 + 一句话摘要 + 链接
- archive 文件不删除，只是换个地方存

---

## 检索优先级

1. **MEMORY.md** — 常驻核心，权重最高
2. **最近几天的 daily 文件** — memory_search 语义检索
3. **archive index** — 当以上路径搜不到时，检索 index 文件
4. **archive 原文** — index 命中后通过链接查阅

---

## 写入时机

- **每日对话结束时**（你说"晚安"或`/new`）→ 写入当天 daily 日志
- **每月1号早上9点** → 自动归档 + 生成 index（系统 cron，无需手动触发）

---

## 设计原则

- **容量可控**：MEMORY.md 是精炼索引，不堆内容；daily 文件按月归档
- **检索优先**：日常只查高频区域，archive 是兜底
- **零额外负担**：归档和 index 生成全自动，海宁不需要任何操作
