# 基于Python QMT的选股回测系统方案

## 项目需求概述
**核心目标**：使用AI编码工具（Vibe Coding）构建基于Python QMT的选股回测系统

### 功能要求
1. **选股条件筛选**：
   - 行业筛选：如"移动互联网服务"
   - 技术指标：最近5天内换手率>5%的次数>0，5日日均换手率>2%
   - 财务指标：
     - 2025年中报扣非净利润>0
     - 2025年三季报扣非净利润>2024年三季报净利润
     - 2025年三季报营业收入>2024年三季报营业收入
     - 2024年扣非净利润>1亿
     - 2024年现金流>1亿

2. **系统架构**：
   - 支持历史数据回测
   - 程序输出备选股供参考
   - 插件化条件封装：用户可自定义条件，易于扩展

3. **AI编码要求**：
   - 基本实现一键生成
   - 能解读生成代码的逻辑，便于理解与改进
   - 同时支持国产和国外AI编码工具方案

### 技术背景
- **QMT**：量化交易平台，支持Python接口
- **Vibe Coding**：AI辅助编程，根据需求描述生成代码

### QMT选股回测系统架构
```python
# 建议的系统模块划分
1. data_provider.py     # QMT数据接口封装
2. condition_engine.py  # 选股条件引擎（插件化核心）
3. backtest_engine.py   # 回测引擎
4. plugin_interface.py  # 插件接口定义
5. main.py             # 主程序
```

### 插件化条件封装示例
```python
# 插件接口
class ConditionPlugin:
    def evaluate(self, stock_data) -> bool:
        """评估单只股票是否满足条件"""
        pass
    
    def get_description(self) -> str:
        """返回条件描述"""
        pass

# 具体条件实现示例
class LiquidityCondition(ConditionPlugin):
    def __init__(self, turnover_rate_threshold=0.05):
        self.threshold = turnover_rate_threshold
    
    def evaluate(self, stock_data):
        # 检查最近5天换手率>5%的次数
        turnover_series = stock_data['turnover_rate_last_5_days']
        return (turnover_series > self.threshold).sum() > 0
    
    def get_description(self):
        return f"最近5天内换手率>{self.threshold*100}%的次数>0"
```

## Claude Code、Code Buddy、Trae 详细分析

### Claude Code (Anthropic)
**本质**：Claude模型家族的代码生成专用模式，并非独立产品
- **访问方式**：通过Claude API调用，或集成在Claude桌面/网页应用中
- **核心优势**：
  1. **推理能力强**：Claude 3.7系列以逻辑推理见长，适合复杂系统设计
  2. **解释详细**：生成的代码附带详细注释和逻辑说明
  3. **长上下文**：支持200K tokens，可提供大量QMT示例代码作为参考
  4. **代码质量**：在架构设计和算法实现上表现优异

**与QMT集成适用性**：
- ✅ 擅长理解金融/量化领域术语
- ✅ 能处理复杂条件组合逻辑
- ✅ 插件化架构设计能力强
- ❌ 需要了解QMT API的具体调用方式（需提供示例）

### Code Buddy (国产)
**定位**：专门面向中文开发者的AI编程助手
- **特点推测**（基于同类产品）：
  1. **中文优化**：需求描述、代码注释、变量命名都更符合中文习惯
  2. **本地化集成**：可能深度集成国内开发工具和云服务
  3. **合规性**：数据存储处理符合国内法规
  4. **定制化**：可能针对国内开发场景（如微信小程序、阿里云等）优化

**与QMT集成适用性**：
- ✅ 中文需求描述更准确
- ✅ 网络访问无阻
- ❓ 对QMT/量化领域训练数据可能较少
- ❓ 需要提供更多QMT示例代码作为学习材料

### Trae (国产)
**信息有限**，可能是新兴AI编程工具：
- **推测特点**：
  1. **垂直领域**：可能针对特定行业（如金融、教育）优化
  2. **场景化**：可能有预设的代码模板和场景
  3. **简化操作**：侧重"一键生成"体验
  4. **解释可视化**：可能提供图形化代码逻辑展示

**与QMT集成适用性**：
- ✅ 如果针对金融领域，可能内置相关模板
- ✅ 简化操作，适合快速原型
- ❓ 自定义能力可能有限
- ❓ 需要验证对QMT API的支持程度

### ⚠️ 常见问题与解决

#### 1. Code Buddy理解不了QMT特定API
- **解决方案**：
  1. 先让Code Buddy生成通用的数据获取函数
  2. 手动修改为QMT API调用
  3. 将修改后的代码作为示例提供给Code Buddy学习
  4. 下次生成时，Code Buddy就能理解QMT特定调用了

#### 2. 生成的代码有逻辑错误
- **解决方案**：
  1. 要求Code Buddy解释每一段代码的逻辑
  2. 发现错误后，告知Code Buddy具体哪里错了
  3. 要求重新生成并解释修正的部分
  4. 建立错误日志，避免重复错误

#### 3. 代码风格不符合要求
- **解决方案**：
  1. 提供代码风格示例
  2. 明确告诉Code Buddy："请遵循PEP 8规范，使用中文注释"
  3. 生成后要求进行代码格式化

#### 4. 插件化架构过于复杂
- **解决方案**：
  1. 先实现非插件版本
  2. 成功后，要求Code Buddy"将这个代码改造成插件架构"
  3. 分步骤进行，先改基类，再改具体条件
