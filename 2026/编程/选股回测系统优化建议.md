# 选股回测系统优化建议

## 背景
当前基于Code Buddy开发的选股回测系统已能正常运行，但回测结果显示策略收益与基准创业板指数（399006.SZ）拉不开差距，未能产生明显的超额收益（α）。

## 一、问题根源分析

### 1.1 现有策略的本质
现有配置筛选的是：
- **创业板**股票（行业过滤）
- **换手率活跃**的个股（技术指标）
- **营收、利润、扣非净利润**连续增长的公司（财务指标）

这实质上是在创业板中挑选“优质成长股”——但这些股票往往已纳入创业板指数，且其中优质公司的权重可能更高，导致策略表现与指数趋同。

### 1.2 缺少的α因子
当前策略忽略了：
- **估值水平**（PE、PB是否合理）
- **动量特征**（是否处于上升趋势的回调阶段）
- **规模因素**（避免过小盘股或已被指数过度包含的大盘股）
- **波动率控制**（避免过度炒作的标的）

### 1.3 调仓策略可能存在的问题
- **10天再平衡周期**可能导致过度交易，增加摩擦成本
- 统一的**12%止损/止盈**可能不符合个股特性
- 缺乏**趋势跟踪**和**移动止损**机制

## 二、改进方向建议

### 2.1 增加α因子筛选（最重要）
在现有财务+换手率筛选基础上，增加1-2个额外条件：

#### 建议增加的筛选维度：
1. **估值筛选**：PE<30，PB<3（避免高位接盘）
2. **动量筛选**：过去3个月涨幅排名前30%，但过去1个月回调5-15%（动量回踩）
3. **规模筛选**：市值50~200亿（避开小盘垃圾股和已被充分定价的大盘权重股）
4. **波动率筛选**：60日波动率<30%（避免情绪过度驱动的标的）

#### 实施优先级（建议按顺序尝试）：
1. 先加估值筛选（PE/PB）→ 看是否能提升收益风险比
2. 再加动量筛选（3个月强但1个月弱）→ 看是否能把握趋势中继机会
3. 最后加规模筛选（50-200亿）→ 避免市值极端的股票

### 2.2 强化调仓策略
```
现有：rebalance_days,10 （每10个交易日调仓）
建议：rebalance_days,22 （按每月约22个交易日调仓，降低交易频率）
```

**更优方案**：
1. 只在有明确卖出信号（止损/止盈触发）时平仓
2. 保留盈利仓位，让利润奔跑
3. 对新选出的股票，只在现有仓位空出资金时买入

### 2.3 加强风险控制
```
# 当前配置：
stop_loss,12.0
take_profit,12.0

# 建议调整：
stop_loss,8.0      # 更严格的止损，避免深套
take_profit,20.0   # 更大的止盈空间，让盈利奔跑
```

**可选增加的机制**：
- **移动止损（Trailing Stop）**：当盈利达到10%后，启动8%的回撤止损
- **分批止盈**：在盈利15%、20%、25%时分别卖出1/3仓位

## 三、具体实施建议

### 3.1 配置文件修改（buy_config.txt扩展建议）
```txt
# 在原配置基础上增加：

# 估值筛选（需要先获取PE、PB数据接口）
# pe,<,30
# pb,<,3

# 市值筛选（单位：亿元）
# market_cap,50,200

# 动量筛选
# momentum,3,30,1,-15  # 过去3个月涨幅前30%，过去1个月最大回撤15%
```

### 3.2 配置文件修改（sell_config.txt扩展建议）
```txt
# 修改风险参数
stop_loss,8.0
take_profit,20.0

# 增加移动止损（可选）
# trailing_stop,10,8

# 调整调仓频率
rebalance_days,22
```

### 3.3 实施步骤
1. **第一步：数据获取** - 确认能否获取PE、PB、市值、动量等额外数据
2. **第二步：单因子测试** - 分别测试每个新增因子的单独效果
3. **第三步：组合测试** - 测试2-3个因子的最佳组合
4. **第四步：参数优化** - 对止损止盈比例、调仓周期进行网格搜索

## 四、可能的风险与注意点

### 4.1 数据可得性
- 部分筛选条件（如PE、PB）需要确认数据源是否可靠、更新及时
- 市值数据要注意是否为流通市值

### 4.2 过拟合风险
- 避免在历史数据上过度优化参数
- 使用**样本外测试**（如用2023年以前数据训练，2024年数据测试）

### 4.3 市场风格变化
- 成长股策略在流动性收紧时可能失效
- 考虑增加**市场环境判断**，在不同市况下使用不同参数

## 五、与现有系统的结合

### 5.1 五维买入评估系统
建议将现有的**五维评估系统**（趋势、基本面、情绪、价格、CS500）与回测系统结合：
- 在选股时计算每只候选股票的**五维评分**
- 只买入评分>=6.0的股票，并根据评分分配仓位
- 记录不同评分区间的实际收益率，验证系统有效性

### 5.2 实施优先级
1. **高优先级**：调整止损止盈参数 + 延长调仓周期
2. **中优先级**：增加市值筛选 + 动量筛选
3. **低优先级**：实现完整的五维评估系统集成

## 六、总结

当前策略的核心问题是**同质化**——在创业板中选优质成长股，这些股票本身构成了创业板指数的主体。想要获得超额收益，必须引入差异化的筛选标准：

1. **估值保护**：避免为成长支付过高溢价
2. **动量加持**：利用趋势的回调上车
3. **规模优化**：在中等市值区间寻找机会
4. **风控强化**：截断亏损，让利润奔跑

建议从最简单的参数调整开始（止损8%、止盈20%、22天调仓），逐步增加更复杂的筛选维度。
